Vaara

Runtime action governance for AI agents — an in-process policy backend for the EU AI Act era.

AI governance tools audit models. Vaara governs actions. A model is scored once at deployment. An agent keeps acting at runtime and modifies regulated state through every tool call. Individual actions may be safe, but sequences can be catastrophic. Vaara is the in-process gate that classifies, scores, decides, and audits every one of them.

The Primitive

Every agent action hits one function. It classifies the action, scores it with a distribution-free confidence interval, decides allow / escalate / deny, and writes every step to a hash-chained audit trail. Outcome feedback flows back so the scorer adapts online, no retraining loop, no threshold tuning. 

from vaara.pipeline import InterceptionPipeline

pipeline = InterceptionPipeline()

result = pipeline.intercept(
    agent_id="treasury-bot",
    tool_name="tx.transfer",
    parameters={"to": "0xabc", "amount": 1000},
)

if result.allowed:
    execute_transfer(...)
    pipeline.report_outcome(result.action_id, outcome_severity=0.0)
else:
    handle_intervention(result)   # decision: "deny" | "escalate"
                                   # risk_interval: (lower, upper)

Why Library, Not SaaS

Runtime gates cannot sit behind a network call. If the only thing stopping a risky action is a round-trip to another vendor's service, the action either happens before the answer arrives, or the agent stalls on latency. Vaara runs in-process in the same Python process that calls the tool. The audit trail is local and portable. Export and sign the trail before handing it to the regulator.
